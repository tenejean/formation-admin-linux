Jour 04 - Terminal et commandes de base

Objectifs d'apprentissage

À la fin de cette journée, les participants seront capables de :

  • Comprendre le concept de terminal et de shell
  • Utiliser les commandes de base pour naviguer dans le système de fichiers
  • Manipuler des fichiers et des répertoires
  • Exécuter des commandes avec des options et des arguments
  • Comprendre le système d'aide et de documentation intégré

Prérequis

  • Connaissances de base sur Linux (Jours 1-3)
  • Installation fonctionnelle d'une distribution Linux

1. Introduction au terminal Linux

1.1 Qu'est-ce que le terminal ?

  • Définition et histoire du terminal
  • Terminal physique vs émulateur de terminal
  • L'importance du terminal dans l'administration Linux
  • Applications courantes des émulateurs de terminal selon les environnements de bureau

1.2 Qu'est-ce que le shell ?

  • Différence entre terminal et shell
  • Rôle du shell : interpréteur de commandes

Shells courants :

  • Bash (Bourne Again SHell) - par défaut sur la plupart des distributions
  • Zsh (Z Shell) - features avancées, par défaut sur macOS récent
  • Fish - shell convivial et intuitif
  • sh, ksh, dash, etc.
Comment identifier votre shell actuel :
echo $SHELL

1.3 Structure d'une commande dans le terminal

commande [options] [arguments]
  • Commande : programme ou commande interne du shell à exécuter
  • Options : modifient le comportement de la commande
    • Format court : préfixé d'un tiret simple (-a, -l)
    • Format long : préfixé de deux tirets (--all, --list)
  • Arguments : cibles ou objets sur lesquels la commande agit

Exemple :

ls -la /home
# ls = commande
# -la = options (-l et -a combinées)
# /home = argument

2. Naviguer dans le système de fichiers

2.1 Structure des répertoires Linux

Hiérarchie du système de fichiers (FHS - Filesystem Hierarchy Standard)

Répertoires principaux et leur rôle :

  • / : racine du système
  • /home : répertoires personnels des utilisateurs
  • /etc : fichiers de configuration
  • /var : données variables (logs, mails, etc.)
  • /usr : programmes et données en lecture seule
  • /bin, /sbin : commandes binaires essentielles
  • /lib : bibliothèques partagées
  • /tmp : fichiers temporaires
  • /boot : fichiers de démarrage
  • /dev : périphériques
  • /proc, /sys : systèmes de fichiers virtuels (informations du noyau)
  • /mnt, /media : points de montage

2.2 Commandes de navigation de base

pwd (Print Working Directory)

Affiche le répertoire de travail actuel

pwd
# Résultat : /home/utilisateur

ls (List)

Lister les fichiers et répertoires

ls # liste basique
ls -l # format long (détails)
ls -a # montre les fichiers cachés (commençant par .)
ls -la # combinaison de -l et -a
ls -lh # format lisible (tailles en Ko, Mo, etc.)

Format long (ls -l) expliqué :

drwxr-xr-x 2 utilisateur groupe 4096 May 10 14:30 Documents
  • d : type (d=répertoire, -=fichier, l=lien symbolique)
  • rwxr-xr-x : permissions
  • 2 : nombre de liens durs
  • utilisateur : propriétaire
  • groupe : groupe propriétaire
  • 4096 : taille (en octets)
  • May 10 14:30 : date de dernière modification
  • Documents : nom

cd (Change Directory)

Changer de répertoire

cd /chemin/absolu # chemin depuis la racine
cd chemin/relatif # chemin depuis la position actuelle
cd ~ # répertoire personnel
cd .. # répertoire parent
cd - # répertoire précédent

2.3 Chemins absolus et relatifs

  • Chemin absolu : commence par /, indépendant de la position actuelle
    Exemple : /home/utilisateur/Documents
  • Chemin relatif : relatif au répertoire de travail actuel
    Exemple : Documents (depuis /home/utilisateur)

Raccourcis :

  • . : répertoire courant
  • .. : répertoire parent
  • ~ : répertoire personnel de l'utilisateur

3. Manipulation de fichiers et répertoires

3.1 Création et suppression de répertoires

mkdir (Make Directory)

mkdir dossier # crée un seul répertoire
mkdir -p parent/enfant/petit # crée des répertoires imbriqués

rmdir (Remove Directory)

rmdir dossier # supprime un répertoire (s'il est vide)

3.2 Création, copie et déplacement de fichiers

touch

touch fichier.txt # crée un fichier vide ou met à jour l'horodatage

cp (Copy)

cp source destination # copie un fichier
cp -r source destination # copie récursive (pour répertoires)
cp -a source destination # préserve les attributs lors de la copie

mv (Move)

mv ancien nouveau # renomme un fichier/répertoire
mv fichier /chemin/destination # déplace un fichier
mv dossier/* destination/ # déplace le contenu d'un dossier

3.3 Suppression de fichiers

rm (Remove)

rm fichier # supprime un fichier
rm -f fichier # force la suppression sans confirmation
rm -r dossier # suppression récursive (pour répertoires)
rm -rf dossier # suppression récursive forcée
⚠️ Attention : rm -rf est dangereux, surtout avec des chemins absolus ou des jokers !

3.4 Visualisation du contenu des fichiers

cat (Concatenate)

cat fichier.txt # affiche le contenu
cat -n fichier.txt # affiche avec numéros de ligne

less

less fichier.txt # navigation avec flèches, Page Up/Down

Commandes utiles dans less :

  • q : quitter
  • /texte : rechercher
  • n : occurrence suivante
  • N : occurrence précédente

head et tail

head fichier.txt # affiche les 10 premières lignes
head -n 20 fichier.txt # affiche les 20 premières lignes
tail fichier.txt # affiche les 10 dernières lignes
tail -n 20 fichier.txt # affiche les 20 dernières lignes
tail -f fichier.log # suit l'évolution du fichier en temps réel

4. Commandes d'information système

4.1 Informations sur le système

uname

uname -a # toutes les informations
uname -r # version du noyau

hostname

hostname # affiche le nom d'hôte

date

date # affiche la date et l'heure actuelles

4.2 Informations sur l'utilisation des ressources

free

free # utilisation de la mémoire
free -h # format "human-readable"

df (Disk Free)

df # utilisation des systèmes de fichiers
df -h # format "human-readable"

du (Disk Usage)

du # taille de tous les sous-répertoires
du -h # format "human-readable"
du -sh */ # taille des sous-répertoires du répertoire courant

top et htop

top # vue dynamique des processus
htop # version améliorée de top (installation peut être nécessaire)

5. Recherche et manipulation de texte

5.1 Recherche de fichiers

find

find /chemin -name "modèle" # recherche par nom
find /home -name "*.txt" # recherche de fichiers .txt dans /home
find /var -type f -size +100M # fichiers de plus de 100 Mo
find /tmp -mtime -7 # fichiers modifiés dans les 7 derniers jours

locate

locate fichier.txt # recherche rapide
sudo updatedb # mise à jour de la base de données

5.2 Recherche dans le contenu des fichiers

grep

grep "motif" fichier # recherche simple
grep -i "motif" fichier # insensible à la casse
grep -r "motif" /chemin # recherche récursive
grep -v "motif" fichier # inverse (lignes ne contenant PAS le motif)
grep -n "motif" fichier # affiche les numéros de ligne

5.3 Redirections et pipelines

Redirection de sortie

commande > fichier # redirige et écrase le fichier
commande >> fichier # redirige et ajoute à la fin du fichier

Redirection d'entrée

commande < fichier # utilise le fichier comme entrée

Redirection d'erreurs

commande 2> erreurs.log # redirige les erreurs vers un fichier
commande > sortie.log 2>&1 # redirige la sortie et les erreurs

Pipelines

commande1 | commande2 # envoie la sortie de commande1 à l'entrée de commande2
cat fichier | grep "motif" # filtre les lignes contenant "motif"
ls -la | less # affiche les résultats de ls page par page

7. Manipulation de base des permissions

7.1 Comprendre les permissions Linux

Type de permission Symbole Description
Lecture r Permet de lire le contenu
Écriture w Permet de modifier le contenu
Exécution x Permet d'exécuter le fichier
Catégorie d'utilisateur Symbole Description
Propriétaire u Utilisateur propriétaire du fichier
Groupe g Groupe propriétaire du fichier
Autres o Tous les autres utilisateurs

7.2 Commandes de gestion des permissions

chmod (Change Mode)

chmod u+x fichier # ajoute l'exécution pour le propriétaire
chmod g-w fichier # retire l'écriture pour le groupe
chmod o=r fichier # définit les droits "autres" à lecture seule
chmod 755 fichier # définit rwx pour propriétaire, r-x pour groupe et autres

chown (Change Owner)

sudo chown utilisateur fichier # change le propriétaire
sudo chown utilisateur:groupe fichier # change propriétaire et groupe

chgrp (Change Group)

sudo chgrp groupe fichier # change le groupe

7.3 Représentation numérique des permissions

Système octal :

  • 4 = read (r)
  • 2 = write (w)
  • 1 = execute (x)
  • 0 = aucun droit

Exemples :

  • chmod 755 fichier = rwxr-xr-x
  • chmod 644 fichier = rw-r--r--
  • chmod 700 fichier = rwx------

Exercices pratiques

Exercice 1 : Navigation et manipulation de base

  1. Ouvrez un terminal
  2. Naviguez jusqu'à votre répertoire personnel (cd ~)
  3. Créez un répertoire nommé exercices
  4. Entrez dans ce répertoire
  5. Créez un fichier texte nommé notes.txt avec la commande touch
  6. Vérifiez qu'il existe avec ls -l
  7. Revenez au répertoire parent
  8. Listez le contenu du répertoire exercices sans y entrer

Exercice 2 : Manipulation du contenu

  1. Dans le répertoire exercices, créez un fichier liste.txt contenant une liste de tâches (utilisez nano ou un autre éditeur)
  2. Affichez le contenu du fichier avec cat
  3. Faites une copie du fichier nommée liste_backup.txt
  4. Ajoutez une nouvelle tâche à la fin du fichier original en utilisant >> (redirection)
  5. Vérifiez les différences entre les deux fichiers

Exercice 3 : Recherche et filtrage

  1. Créez un nouveau répertoire nommé recherche
  2. Dans ce répertoire, créez plusieurs fichiers texte avec différents contenus
  3. Utilisez grep pour trouver tous les fichiers contenant un mot spécifique
  4. Essayez des recherches plus complexes en combinant les commandes avec des pipelines

Exercice 4 : Permissions

  1. Créez un fichier script simple nommé hello.sh contenant :
    #!/bin/bash
    echo "Hello, Linux!"
  2. Essayez de l'exécuter directement (./hello.sh) et observez l'erreur
  3. Changez les permissions pour le rendre exécutable
  4. Exécutez-le à nouveau

Résumé des points clés

  • Le terminal Linux offre un contrôle puissant et précis sur le système
  • Les commandes de base permettent de naviguer et manipuler les fichiers efficacement
  • Les redirections et pipelines permettent de combiner les commandes de manière puissante
  • Le système de permissions Linux contrôle l'accès aux fichiers et répertoires
  • La documentation intégrée (man, help, info) est une ressource essentielle
