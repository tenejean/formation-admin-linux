1. Introduction à la gestion des logiciels sous Linux

1.1 Particularités de Linux pour la gestion logicielle

Contrairement aux systèmes Windows où l'installation se fait généralement via des fichiers .exe ou .msi, Linux utilise des systèmes de gestion de paquets centralisés qui offrent plusieurs avantages :

  • Centralisation : tous les logiciels sont disponibles depuis des dépôts centraux
  • Dépendances : gestion automatique des bibliothèques et composants requis
  • Mises à jour : système unifié de mise à jour pour tous les logiciels
  • Sécurité : vérification d'intégrité et d'authenticité des paquets
  • Désinstallation propre : suppression complète des composants installés

1.2 Terminologie importante

  • Paquet : archive contenant les fichiers d'un logiciel et ses métadonnées
  • Dépendance : relation entre paquets où l'un nécessite l'autre pour fonctionner
  • Dépôt : serveur contenant une collection de paquets disponibles à l'installation
  • Gestionnaire de paquets : outil qui automatise l'installation, la mise à jour et la suppression des paquets
  • Miroir : serveur qui duplique le contenu d'un dépôt officiel

2. Les principaux systèmes de gestion de paquets

2.1 Famille Debian (Debian, Ubuntu, Linux Mint...)

2.1.1 Système dpkg et APT

  • dpkg : utilitaire de bas niveau pour manipuler les paquets .deb
  • APT (Advanced Package Tool) : système de haut niveau qui gère les dépendances et les dépôts
  • apt : interface simplifiée et moderne remplaçant apt-get et apt-cache
  • aptitude : interface alternative à APT (mode texte ou ligne de commande)

2.1.2 Structure d'un paquet .deb

Archives au format ar contenant:

  • debian-binary : version du format
  • control.tar.gz : métadonnées (dépendances, descriptions...)
  • data.tar.xz : fichiers à installer sur le système

2.2 Famille Red Hat (RHEL, Fedora, CentOS...)

2.2.1 Système RPM et DNF/YUM

  • RPM (Red Hat Package Manager) : utilitaire de bas niveau pour les paquets .rpm
  • YUM (Yellowdog Updater Modified) : gestionnaire de haut niveau (plus ancien)
  • DNF (Dandified YUM) : remplaçant moderne de YUM (depuis Fedora 22 et RHEL 8)

2.2.2 Structure d'un paquet .rpm

Format binaire spécifique contenant:

  • En-tête (métadonnées)
  • Archive compressée contenant les fichiers
  • Scripts de pré/post installation

2.3 Autres gestionnaires notables

  • Pacman : utilisé par Arch Linux et dérivés
  • Zypper : utilisé par SUSE et openSUSE
  • Portage : système basé sur les sources pour Gentoo
  • Alpine's apk : gestionnaire léger utilisé dans Alpine Linux

2.4 Gestionnaires universels

  • Flatpak : format de paquets indépendant des distributions
  • Snap : système développé par Canonical (Ubuntu)
  • AppImage : applications portables ne nécessitant pas d'installation

3. Installation de logiciels avec APT (Debian/Ubuntu)

3.1 Configuration des dépôts

Les dépôts sont configurés dans les fichiers suivants :

  • /etc/apt/sources.list : dépôts principaux
  • /etc/apt/sources.list.d/ : dépôts additionnels (fichiers .list)

Structure d'une ligne de dépôt :

deb [options] URI distribution composants

Exemple :

deb http://archive.ubuntu.com/ubuntu/ jammy main restricted universe multiverse

3.2 Mise à jour de la liste des paquets

Cette commande synchronise les métadonnées des paquets disponibles avec les dépôts configurés.

sudo apt update

3.3 Installation de logiciels

sudo apt install nom_du_paquet

Options utiles :

  • -y : répondre automatiquement "oui" aux questions
  • --no-install-recommends : ne pas installer les paquets recommandés
  • --reinstall : réinstaller un paquet déjà installé

Installation de plusieurs paquets :

sudo apt install paquet1 paquet2 paquet3

3.4 Mise à jour des logiciels

Mettre à jour tous les paquets installés :

sudo apt upgrade

Mettre à jour la distribution (peut installer/supprimer des paquets) :

sudo apt dist-upgrade
# ou la commande moderne équivalente sudo apt full-upgrade

3.5 Suppression de logiciels

# Supprimer un paquet sudo apt remove nom_du_paquet # Supprimer un paquet et ses fichiers de configuration sudo apt purge nom_du_paquet # ou sudo apt remove --purge nom_du_paquet # Supprimer les paquets installés automatiquement et devenus inutiles sudo apt autoremove

3.6 Recherche de paquets

# Rechercher dans les noms et descriptions apt search terme_recherche # Afficher les informations détaillées d'un paquet apt show nom_du_paquet # Lister les paquets installés apt list --installed # Lister les paquets pouvant être mis à jour apt list --upgradable

4. Installation de logiciels avec DNF/YUM (RHEL/Fedora/CentOS)

4.1 Configuration des dépôts

Les dépôts sont configurés dans les fichiers suivants :

  • /etc/yum.repos.d/ : répertoire contenant les fichiers .repo

Structure d'un fichier .repo :

[id_du_depot] name=Description du dépôt baseurl=URL_du_dépôt enabled=1 gpgcheck=1 gpgkey=URL_de_la_clé_GPG

4.2 Mise à jour de la liste des paquets

Avec DNF, la mise à jour de la liste est automatique à chaque commande, mais peut être forcée :

sudo dnf makecache

4.3 Installation de logiciels

sudo dnf install nom_du_paquet

Options utiles :

  • -y : répondre automatiquement "oui" aux questions
  • --nogpgcheck : ne pas vérifier les signatures GPG (déconseillé)
  • --refresh : rafraîchir les métadonnées avant l'installation

4.4 Mise à jour des logiciels

# Mettre à jour tous les paquets sudo dnf upgrade # Alternative plus ancienne (YUM) sudo yum update

4.5 Suppression de logiciels

# Supprimer un paquet sudo dnf remove nom_du_paquet # Supprimer les paquets devenus inutiles (dépendances orphelines) sudo dnf autoremove

4.6 Recherche de paquets

# Rechercher dans les noms et descriptions sudo dnf search terme_recherche # Afficher les informations d'un paquet sudo dnf info nom_du_paquet # Lister les paquets installés sudo dnf list installed # Lister les paquets disponibles pour mise à jour sudo dnf check-update

4.7 Groupes de paquets

DNF/YUM permet d'installer des ensembles prédéfinis de paquets :

# Lister les groupes disponibles sudo dnf group list # Installer un groupe sudo dnf group install "Nom du groupe"

5. Gestion des paquets avec Pacman (Arch Linux)

5.1 Mise à jour de la liste des paquets

sudo pacman -Sy

5.2 Installation de logiciels

sudo pacman -S nom_du_paquet

5.3 Mise à jour des logiciels

# Synchroniser la base de données et mettre à jour tous les paquets sudo pacman -Syu

5.4 Suppression de logiciels

# Supprimer un paquet sudo pacman -R nom_du_paquet # Supprimer un paquet et ses dépendances non utilisées par d'autres paquets sudo pacman -Rs nom_du_paquet

5.5 Recherche de paquets

# Rechercher dans les noms et descriptions sudo pacman -Ss terme_recherche # Afficher les informations d'un paquet sudo pacman -Si nom_du_paquet # Lister les paquets installés sudo pacman -Q

6. Systèmes de paquets universels

6.1 Flatpak

Conçu pour fournir des applications isolées du système sous-jacent.

# Installation de Flatpak sudo apt install flatpak # Sur Debian/Ubuntu sudo dnf install flatpak # Sur Fedora # Ajouter le dépôt Flathub flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo # Installer une application flatpak install flathub org.nom.Application # Exécuter une application flatpak run org.nom.Application # Mettre à jour toutes les applications Flatpak flatpak update # Supprimer une application flatpak uninstall org.nom.Application

6.2 Snap

Développé par Canonical, utilisé principalement sous Ubuntu.

# Installation de Snap (préinstallé sur Ubuntu) sudo apt install snapd # Sur Debian/Ubuntu sudo dnf install snapd # Sur Fedora # Installer une application sudo snap install nom_application # Lister les snaps installés snap list # Mettre à jour un snap sudo snap refresh nom_application # Mettre à jour tous les snaps sudo snap refresh # Supprimer un snap sudo snap remove nom_application

6.3 AppImage

Applications autonomes qui ne nécessitent pas d'installation.

Utilisation:

  1. Télécharger le fichier AppImage
  2. Rendre le fichier exécutable: chmod +x fichier.AppImage
  3. Exécuter: ./fichier.AppImage

7. Installation depuis les sources

7.1 Pourquoi compiler depuis les sources?

  • Logiciel non disponible dans les dépôts officiels
  • Besoin de personnalisation avancée
  • Optimisations spécifiques à votre matériel
  • Versions plus récentes que celles disponibles dans les dépôts

7.2 Prérequis pour la compilation

Installation des outils de développement nécessaires:

Sur Debian/Ubuntu:

sudo apt install build-essential autoconf automake libtool pkg-config

Sur RHEL/Fedora/CentOS:

sudo dnf group install "Development Tools"

Sur Arch Linux:

sudo pacman -S base-devel

7.3 Processus typique de compilation

# 1. Télécharger et extraire les sources wget https://example.com/logiciel-1.0.tar.gz tar xzf logiciel-1.0.tar.gz cd logiciel-1.0 # 2. Configurer (vérifie les dépendances et crée le Makefile) ./configure --prefix=/usr/local # 3. Compiler make # 4. Installer sudo make install

7.4 Gestion des dépendances lors de la compilation

Sur Debian/Ubuntu, installer les bibliothèques de développement:

sudo apt install libxml2-dev libssl-dev # exemple de dépendances courantes

7.5 Utilisation de checkinstall

checkinstall crée un paquet système à partir d'une installation depuis les sources, facilitant la désinstallation ultérieure:

sudo apt install checkinstall # Sur Debian/Ubuntu # Au lieu de 'make install', utilisez: sudo checkinstall

8. Systèmes de gestion de versions pour développeurs

8.1 pip (Python)

# Installation de pip sudo apt install python3-pip # Debian/Ubuntu sudo dnf install python3-pip # Fedora/RHEL # Installer un paquet Python pip install nom_du_paquet # Installer pour l'utilisateur courant uniquement pip install --user nom_du_paquet # Mettre à jour un paquet pip install --upgrade nom_du_paquet # Désinstaller un paquet pip uninstall nom_du_paquet

8.2 npm (Node.js)

# Installation de Node.js et npm sudo apt install nodejs npm # Debian/Ubuntu sudo dnf install nodejs npm # Fedora/RHEL # Installer un paquet globalement sudo npm install -g nom_du_paquet # Installer en local (dans le projet) npm install nom_du_paquet # Mettre à jour un paquet npm update nom_du_paquet # Désinstaller un paquet npm uninstall nom_du_paquet

8.3 gem (Ruby)

# Installation de Ruby et gem sudo apt install ruby-full # Debian/Ubuntu sudo dnf install ruby # Fedora/RHEL # Installer une gem gem install nom_de_la_gem # Mettre à jour toutes les gems gem update # Désinstaller une gem gem uninstall nom_de_la_gem

9. Bonnes pratiques et dépannage

9.1 Bonnes pratiques

  • Privilégier les paquets des dépôts officiels quand c'est possible
  • Mettre régulièrement à jour le système pour des raisons de sécurité
  • Faire attention aux sources tierces (PPA, COPR, etc.)
  • Documenter les installations manuelles et personnalisations
  • Faire des sauvegardes avant les mises à jour majeures

9.2 Résolution des dépendances brisées

Sur Debian/Ubuntu:

# Réparer les paquets cassés sudo apt --fix-broken install # Forcer la reconfiguration des paquets sudo dpkg --configure -a

Sur RHEL/Fedora/CentOS:

# Nettoyer les métadonnées et le cache sudo dnf clean all

9.3 Problèmes courants et solutions

Erreur de verrouillage

# Sur Debian/Ubuntu sudo rm /var/lib/apt/lists/lock sudo rm /var/cache/apt/archives/lock sudo rm /var/lib/dpkg/lock*

Conflit de paquets

Examiner soigneusement les messages d'erreur et:

  1. Désinstaller le paquet conflictuel
  2. Chercher une version alternative
  3. Utiliser des options de forçage (avec prudence)

Corruption de la base de données de paquets

sudo apt clean sudo apt update --fix-missing

Conclusion

L'installation et la gestion des logiciels sont des compétences fondamentales pour tout administrateur système Linux. Les différentes distributions proposent des outils variés, mais reposant sur des principes similaires. La compréhension des systèmes de gestion de paquets permet non seulement d'installer les logiciels nécessaires, mais aussi de maintenir le système à jour et sécurisé. Dans le prochain cours, nous aborderons la gestion des processus et services, qui sont les composants actifs utilisant les logiciels installés.
