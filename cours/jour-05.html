1. Introduction aux systèmes de fichiers Linux

Qu'est-ce qu'un système de fichiers?

Un système de fichiers est une méthode pour stocker et organiser les fichiers sur un support de stockage comme un disque dur, une clé USB ou un SSD. Il définit comment les données sont stockées, nommées, accédées et protégées sur le support physique.

Principe fondamental : Sous Linux, le système de fichiers suit le principe "tout est fichier". Cela signifie que non seulement les documents et programmes sont représentés comme des fichiers, mais également les périphériques matériels, les processus et les communications réseau.

Caractéristiques des systèmes de fichiers Linux

  • Structure hiérarchique : Organisation en arborescence à partir d'un répertoire racine (/)
  • Sensible à la casse : Contrairement à Windows, Linux distingue les majuscules des minuscules dans les noms de fichiers
  • Pas d'utilisation de lettres de lecteurs : Tous les périphériques sont montés dans une arborescence unique
  • Pas d'extension obligatoire : Les extensions de fichiers ne sont pas obligatoires mais souvent utilisées par convention
  • Métadonnées riches : Stockage d'informations détaillées sur chaque fichier (permissions, propriétaire, date de modification, etc.)

2. Hiérarchie standard du système de fichiers

Linux suit le standard FHS (Filesystem Hierarchy Standard) qui définit l'organisation des répertoires principaux du système.

Répertoires principaux et leurs fonctions

Répertoire Description
/Répertoire racine, point de départ de toute l'arborescence
/binCommandes binaires essentielles pour tous les utilisateurs
/bootFichiers nécessaires au démarrage du système (noyau, bootloader)
/devFichiers spéciaux représentant les périphériques matériels
/etcFichiers de configuration du système
/homeRépertoires personnels des utilisateurs
/libBibliothèques partagées essentielles et modules du noyau
/mediaPoint de montage pour les supports amovibles
/mntPoint de montage temporaire pour les systèmes de fichiers
/optEmplacement pour l'installation de logiciels optionnels
/procSystème de fichiers virtuel contenant des informations sur les processus et le système
/rootRépertoire personnel de l'utilisateur root
/runDonnées d'exécution pour les processus démarrés depuis le boot
/sbinBinaires système essentiels (administration)
/srvDonnées pour les services fournis par le système
/sysSystème de fichiers virtuel exportant les informations du noyau
/tmpFichiers temporaires (généralement effacés au redémarrage)
/usrHiérarchie secondaire pour les données utilisateur en lecture seule
/varDonnées variables (logs, bases de données, sites web, etc.)

Importance de respecter cette hiérarchie

Respecter cette hiérarchie standardisée permet :

  • D'assurer la portabilité entre les distributions Linux
  • De faciliter la maintenance et l'administration
  • D'améliorer la sécurité par la séparation des données
  • De simplifier les sauvegardes et les mises à jour

3. Types de systèmes de fichiers sous Linux

Linux prend en charge de nombreux systèmes de fichiers, chacun avec ses avantages et inconvénients.

Systèmes de fichiers natifs Linux

Ext2/3/4 (Extended File System)

  • Ext2 : Premier système de fichiers robuste pour Linux, mais sans journalisation
  • Ext3 : Amélioration d'Ext2 avec journalisation pour prévenir la corruption des données
  • Ext4 : Version moderne avec support des gros fichiers, meilleure performance, et moins de fragmentation

XFS

  • Développé par SGI, optimisé pour les fichiers volumineux
  • Très performant pour les serveurs et les systèmes de stockage à grande échelle
  • Journalisation avancée et bonnes performances en écriture parallèle

Btrfs (B-tree File System)

Système moderne avec fonctionnalités avancées :

  • Snapshots et rollbacks
  • Compression à la volée
  • RAID logiciel intégré
  • Vérification d'intégrité des données par sommes de contrôle

Systèmes de fichiers pour la compatibilité

  • FAT32 : Compatible avec presque tous les systèmes, mais limité (fichiers max 4GB)
  • NTFS : Système de fichiers Windows, bien supporté en lecture sous Linux
  • exFAT : Pour les périphériques de stockage externes compatibles avec Windows et macOS

Systèmes de fichiers spéciaux

  • tmpfs : Système de fichiers en RAM pour les données temporaires
  • procfs : Système virtuel exposant les informations sur les processus via /proc
  • sysfs : Représente les périphériques et bus du système via /sys
  • NFS : Pour le partage de fichiers sur réseau

4. Navigation dans le système de fichiers

Commandes de base

Commande Description Exemples
pwd Affiche le répertoire de travail actuel pwd
cd Change de répertoire cd /etc, cd ~, cd ..
ls Liste le contenu d'un répertoire ls -la, ls -lh /var

Options importantes pour ls

  • -l : Format liste détaillée
  • -a : Affiche les fichiers cachés (commençant par .)
  • -h : Tailles en format lisible (Ko, Mo, Go)
  • -R : Récursif (liste les sous-répertoires)
  • -t : Trie par date de modification
  • -S : Trie par taille de fichier

Astuces de navigation

  • cd - : Retourne au répertoire précédent
  • cd ~ : Accède au répertoire personnel
  • cd .. : Remonte d'un niveau dans l'arborescence
  • cd / : Accède au répertoire racine
  • Utilisation des Tab pour l'auto-complétion

5. Manipulation des fichiers et répertoires

Création et suppression

Commande Description Exemples
mkdir Crée un répertoire mkdir dossier, mkdir -p parent/enfant
touch Crée un fichier vide ou met à jour la date touch fichier.txt
rm Supprime un fichier rm fichier.txt, rm -rf dossier
rmdir Supprime un répertoire vide rmdir dossier

Copie et déplacement

Commande Description Exemples
cp Copie des fichiers ou répertoires cp source.txt destination.txt, cp -r dossier1 dossier2
mv Déplace ou renomme des fichiers mv ancien.txt nouveau.txt, mv fichier.txt /tmp/

Affichage du contenu

Commande Description Exemples
cat Affiche tout le contenu d'un fichier cat fichier.txt
less Affiche le contenu page par page less fichier.txt
head Affiche les premières lignes head -n 10 fichier.txt
tail Affiche les dernières lignes tail -n 20 fichier.txt, tail -f /var/log/syslog

Options importantes

  • cp -r : Copie récursivement (pour les répertoires)
  • rm -r : Supprime récursivement
  • rm -f : Force la suppression sans confirmation
  • mv -i : Mode interactif (demande confirmation)
  • tail -f : Suit les modifications du fichier en temps réel (très utile pour les logs)

6. Liens (symboliques et physiques)

Les liens sont des méthodes pour référencer des fichiers à partir de plusieurs emplacements.

Liens physiques (hard links)

  • Créés avec ln fichier_source lien_physique
  • Pointent directement vers l'inode (structure de données contenant les métadonnées) du fichier
  • Limitations :
    • Ne peuvent pas pointer vers des répertoires
    • Ne peuvent pas traverser les systèmes de fichiers
    • Ne peuvent pas pointer vers des fichiers inexistants

Liens symboliques (soft links)

  • Créés avec ln -s fichier_source lien_symbolique
  • Équivalent aux raccourcis sous Windows
  • Avantages :
    • Peuvent pointer vers des répertoires
    • Peuvent traverser les systèmes de fichiers
    • Peuvent pointer vers des fichiers inexistants (liens cassés)

Différences clés

Caractéristique Lien physique Lien symbolique
Suppression de l'original Le contenu reste accessible Le lien devient inutilisable
Taille Même taille que l'original Petite taille (chemin d'accès)
Inode Même inode que l'original Inode différent
Traversée de systèmes Non Oui
Pointer vers un répertoire Non Oui

Utilisation des liens

Les liens sont particulièrement utiles pour :

  • Accéder à un même fichier depuis différents emplacements
  • Créer des versions alternatives de configuration
  • Gérer plusieurs versions de logiciels
  • Économiser de l'espace disque (liens physiques)

7. Montage et démontage des systèmes de fichiers

Le montage est le processus qui consiste à rendre un système de fichiers accessible dans l'arborescence Linux.

Commandes de base

Commande Description Exemples
mount Monte un système de fichiers mount /dev/sdb1 /mnt, mount -t ext4 /dev/sda2 /data
umount Démonte un système de fichiers umount /mnt, umount /dev/sdb1
lsblk Liste les périphériques de bloc lsblk
blkid Affiche les UUID des périphériques blkid
df Affiche l'utilisation de l'espace disque df -h

Fichier /etc/fstab

Le fichier /etc/fstab contient les configurations de montage automatique au démarrage :

# <périphérique> <point de montage> <type> <options> <dump> <pass> UUID=1234-5678 / ext4 defaults 0 1 /dev/sdb1 /data ext4 defaults 0 2

Options de montage courantes

  • defaults : Combine les options rw, suid, dev, exec, auto, nouser, async
  • ro / rw : Lecture seule / lecture-écriture
  • noexec / exec : Interdit/permet l'exécution de binaires
  • auto / noauto : Montage automatique au démarrage ou non
  • user / nouser : Permet/interdit aux utilisateurs ordinaires de monter
  • sync / async : Entrées/sorties synchrones ou asynchrones

Montage de systèmes de fichiers réseau

  • NFS : mount -t nfs serveur:/partage /mnt/nfs
  • CIFS/SMB : mount -t cifs //serveur/partage /mnt/smb -o username=user,password=pass

8. Gestion de l'espace disque

Commandes d'analyse d'espace

Commande Description Exemples
df Affiche l'utilisation des systèmes de fichiers df -h, df -i
du Affiche l'utilisation d'espace par répertoire du -sh /var, du -h --max-depth=1 /
ncdu Version interactive de du (à installer) ncdu /

Options utiles

  • df -h : Affiche les tailles en format lisible (Go, Mo)
  • df -i : Affiche l'utilisation des inodes
  • du -s : Affiche seulement le total
  • du -h : Format lisible
  • du --max-depth=N : Limite la profondeur d'analyse

Outils de gestion d'espace disque

  • quotas : Limiter l'espace utilisé par utilisateur ou groupe
  • logrotate : Rotation automatique des fichiers journaux pour économiser l'espace
  • tmpwatch : Suppression automatique des fichiers temporaires anciens

9. Recherche de fichiers

Commandes principales

Commande Description Exemples
find Recherche puissante et personnalisable find /home -name "*.txt", find / -size +100M
locate Recherche rapide basée sur base de données locate fichier.conf
which Localise les exécutables dans $PATH which python
whereis Trouve les binaires, sources et manuels whereis bash

Options importantes pour find

  • -name pattern : Recherche par nom de fichier
  • -type f|d|l : Type de fichier (f=fichier, d=répertoire, l=lien)
  • -size +/-Nk|M|G : Taille de fichier
  • -user nom : Appartenant à un utilisateur
  • -mtime +/-N : Modifié il y a N jours
  • -exec command {} \; : Exécute une commande sur les résultats

Exemples avancés de recherche

# Trouver les fichiers modifiés ces dernières 24h find /home -mtime -1 # Trouver les fichiers volumineux (+100Mo) find / -type f -size +100M # Trouver et supprimer les fichiers temporaires find /tmp -name "*.tmp" -exec rm {} \; # Trouver les fichiers avec permissions 777 find / -type f -perm 0777

Base de données locate

  • updatedb : Met à jour la base de données de locate
  • Généralement exécuté quotidiennement via cron
  • Plus rapide que find mais moins précis (pas en temps réel)

10. Travaux pratiques

TP 1: Navigation et manipulation de base

  1. Créez un répertoire ~/linux_training avec deux sous-répertoires docs et scripts
  2. Créez un fichier texte notes.txt dans docs contenant quelques lignes
  3. Créez un lien symbolique vers ce fichier dans votre répertoire personnel
  4. Vérifiez l'existence du lien et affichez son contenu
  5. Déplacez le fichier original et observez ce qui arrive au lien

TP 2: Recherche de fichiers

  1. Trouvez tous les fichiers de configuration (*.conf) dans /etc modifiés au cours des 7 derniers jours
  2. Recherchez tous les fichiers appartenant à votre utilisateur et de taille supérieure à 10Mo
  3. Créez une commande qui affiche les 10 plus gros fichiers d'un répertoire donné

TP 3: Analyse d'espace disque

  1. Identifiez les systèmes de fichiers avec moins de 20% d'espace libre
  2. Trouvez les 5 plus grands répertoires sous /var
  3. Vérifiez combien d'inodes sont utilisés sur votre système de fichiers racine

TP 4: Montage et démontage

  1. Créez un fichier image de 100Mo avec dd
  2. Formatez-le en ext4
  3. Montez-le sur un point de montage temporaire
  4. Créez quelques fichiers dedans
  5. Démontez-le proprement
