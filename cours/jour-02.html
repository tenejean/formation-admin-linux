Objectifs d'apprentissage

À la fin de cette journée, les participants seront capables de :

  • Identifier les considérations préalables à l'installation d'un système Linux
  • Préparer un environnement d'installation avec différentes méthodes
  • Réaliser une installation complète d'une distribution Linux
  • Configurer correctement les éléments post-installation essentiels

Prérequis

  • Connaissances de base sur Linux (Jour 1)
  • Notions élémentaires sur le partitionnement de disques
  • Compréhension basique du matériel informatique

1. Préparation à l'installation

1.1 Choix de la distribution

Critères à considérer lors du choix d'une distribution pour un environnement spécifique :

  • Usage prévu : serveur, station de travail, développement, etc.
  • Support matériel : compatibilité avec les composants spécifiques
  • Durée de support : versions LTS (Long Term Support) vs versions standard
  • Environnement de bureau : GNOME, KDE, Xfce, etc. (pour les installations desktop)
  • Stabilité vs nouveauté : distributions conservatrices vs rolling release

Pour cette formation, nous utiliserons principalement :

  • Ubuntu Server ou Debian pour les environnements serveur
  • Ubuntu Desktop pour les environnements graphiques

1.2 Vérification des prérequis matériels

  • Processeur : architecture compatible (x86_64/AMD64, ARM64, etc.)
  • Mémoire RAM minimale :
    • Serveur : 512 MB (minimal), 1 GB (recommandé)
    • Bureau : 2 GB (minimal), 4 GB (recommandé)
  • Espace disque :
    • Serveur : 10 GB minimum
    • Bureau : 20 GB minimum
  • Périphériques : vérification de la compatibilité des cartes réseau, cartes graphiques, etc.

1.3 Planification du partitionnement

Schémas de partitionnement courants :

Schéma Partitions Taille recommandée
Simple / (root) Tout l'espace disponible
Standard /boot 500 MB
swap 1-2× la RAM
/ (root) Reste de l'espace
Avancé /boot 500 MB
swap 1-2× la RAM
/ (root) 20-30 GB
/home Pour les données utilisateurs
/var Pour les logs, caches, etc.
/tmp Pour les fichiers temporaires

Considérations sur les systèmes de fichiers :

  • ext4 : standard, stable et éprouvé
  • XFS : performant pour les gros fichiers
  • Btrfs : moderne avec snapshots et autres fonctionnalités avancées
  • ZFS : avancé avec gestion de volumes intégrée (non inclus par défaut dans certaines distributions)

2. Méthodes d'installation

2.1 Installation physique

Processus pour une installation sur matériel physique :

  1. Téléchargement de l'image ISO depuis le site officiel de la distribution
  2. Vérification de l'intégrité (sommes MD5/SHA)
  3. Création d'un média d'installation (USB, DVD)
  4. Configuration du BIOS/UEFI pour démarrer sur ce média
  5. Démarrage sur le média d'installation

Création d'une clé USB bootable

Outils selon le système d'exploitation hôte :

  • Linux : dd, Etcher, UNetbootin
  • Windows : Rufus, Etcher, UNetbootin
  • macOS : Etcher, dd, UNetbootin

Exemple avec dd sous Linux :

# Remplacer sdX par le périphérique correspondant à votre clé USB sudo dd if=ubuntu-22.04-server-amd64.iso of=/dev/sdX bs=4M status=progress

2.2 Installation virtuelle

Options de virtualisation pour l'apprentissage et les tests :

  • VirtualBox : gratuit, multi-plateforme, facile à utiliser
  • VMware : versions gratuites et payantes, très répandu en entreprise
  • KVM/QEMU : natif sous Linux, performant
  • Hyper-V : natif sous Windows Server et Windows 10/11 Pro

Avantages de la virtualisation pour l'apprentissage :

  • Isolation des systèmes
  • Possibilité de créer des snapshots
  • Facilité de réinitialisation
  • Testez plusieurs distributions sans réinstallation physique

2.3 Installation automatisée

Pour les déploiements à grande échelle :

  • Kickstart (Red Hat, CentOS, Fedora)
  • Preseed (Debian, Ubuntu)
  • AutoYaST (SUSE)
  • Cloud-init (pour les environnements cloud)

4. Configuration post-installation

4.1 Mise à jour du système

# Sur Debian/Ubuntu sudo apt update sudo apt upgrade # Sur RHEL/CentOS/Fedora sudo dnf update

4.2 Installation des pilotes additionnels

  • Pilotes propriétaires vs open source
  • Pilotes graphiques (NVIDIA, AMD)
  • Pilotes pour matériel spécifique (Wi-Fi, imprimantes, etc.)

4.3 Configuration du réseau

  • Attribution d'une adresse IP statique (si nécessaire)
  • Configuration du nom d'hôte
  • Configuration DNS

4.4 Activation/désactivation des services

# Avec systemd sudo systemctl status [service] sudo systemctl enable [service] sudo systemctl disable [service]

4.5 Installation des outils essentiels

Packages de base recommandés pour l'administration :

# Sur Debian/Ubuntu sudo apt install vim htop net-tools dnsutils curl wget git

Exercices pratiques

Exercice 1 : Installation virtuelle

  1. Installez VirtualBox (ou autre solution de virtualisation)
  2. Téléchargez l'ISO d'Ubuntu Server 22.04 LTS
  3. Créez une machine virtuelle avec les paramètres appropriés
  4. Réalisez l'installation complète en suivant les étapes du cours

Exercice 2 : Partitionnement avancé

Dans une seconde machine virtuelle, réalisez une installation avec un schéma de partitionnement avancé :

  • Partition /boot (500 MB)
  • Partition swap (2 GB)
  • Partition / (root) (15 GB)
  • Partition /home (reste de l'espace)

Exercice 3 : Post-installation

Après l'installation :

  1. Mettez à jour le système
  2. Installez les outils essentiels mentionnés dans le cours
  3. Vérifiez l'état des services avec systemctl
  4. Configurez le réseau pour utiliser une adresse IP statique

Résumé des points clés

  • L'installation de Linux nécessite une préparation adéquate (choix de distribution, vérification matérielle)
  • Le partitionnement est une étape cruciale qui impacte la sécurité et les performances
  • La virtualisation offre un environnement sécurisé pour l'apprentissage
  • La configuration post-installation est essentielle pour un système sécurisé et fonctionnel
